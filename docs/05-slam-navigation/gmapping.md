# Gmapping SLAM 建图

> 一边走，一边把世界画出来

## 第一性原理

Gmapping 是一种**基于粒子滤波的 2D SLAM**。

它同时做两件事：

1. **定位**：估计机器人当前位姿
2. **建图**：把雷达看到的障碍物写进栅格地图

为什么要用粒子？因为机器人真实位姿并不确定。Gmapping 会维护很多个 “假设位姿”（粒子），每个粒子都尝试解释当前雷达观测，然后给更合理的粒子更高权重。

你可以把它理解成：

> “机器人同时假装自己在 8 个不同的位置，然后看哪个位置最能解释现在的雷达扫描。”

## 关键参数详解

### `particles = 8`

- 含义：粒子数量
- 作用：表示 Gmapping 同时保留多少种位姿假设
- 当前值：**8**

影响：

- 粒子少：CPU 占用低，运行快，但更容易定位错误
- 粒子多：更稳，但更吃算力

> [!WARNING]
>
> 8 个粒子对真实比赛环境来说偏少，更像 “能跑起来” 的最低配置。赛场如果结构重复、特征少，容易误匹配。

建议：

- 小场地、特征清晰：15 $\sim$ 30
- 场地大、容易重复：30 $\sim$ 50

### `delta = 0.02`

- 含义：地图分辨率，单位 m/格
- 当前值：**0.02**（2cm）

影响：

- 分辨率小（更细）：地图细节更多，但内存占用和更新计算增加
- 分辨率大（更粗）：地图更轻，但细节缺失

2cm 分辨率非常细，对小车和室内竞赛是有帮助的，但代价是 CPU 更忙。

### `minimumScore = 30`

- 含义：接受一次 scan matching 的最低分数
- 当前值：**30**

影响：

- 太低：哪怕匹配得一般也接受，容易把地图拉歪
- 太高：匹配过于苛刻，机器人可能“卡住”不更新

> [!TIP]
>
> 如果你建图时经常出现地图扭曲、重影，先尝试把 `minimumScore` 提高到 50 左右。

### `linearUpdate = 0.05 / angularUpdate = 0.0436`

- 含义：机器人至少移动多少距离 / 转多少角度才更新一次地图
- 当前值：5cm / 2.5°

影响：

- 阈值小：地图更新更频繁，更细致，但计算更重
- 阈值大：更新更稀疏，建图更快，但细节丢失

### `sigma` / `lsigma` / `kernelSize` / `iterations`

这些参数属于 scan matching 内部优化：

| 参数 | 当前值 | 直觉理解 |
|------|--------|----------|
| `sigma` | 0.05 | 观测噪声大小 |
| `lsigma` | 0.075 | 激光匹配中的平滑程度 |
| `kernelSize` | 3 | 局部搜索核大小 |
| `iterations` | 5 | 每次匹配做几轮优化 |

如果你不是在做 SLAM 算法研究，通常不先动这些。优先调 `particles`、`minimumScore`、`delta`。

## 参数如何映射到比赛表现

### 现象 1：地图边缘抖动、重影

可能原因：

- `particles` 太少
- `minimumScore` 太低
- 建图时速度太快

### 现象 2：地图很慢才更新

可能原因：

- `linearUpdate` / `angularUpdate` 太大
- CPU 太忙

### 现象 3：机器人一转弯地图就扭曲

可能原因：

- 轮式里程计不准
- IMU 没校准
- `minimumScore` 太低

## 建图操作建议

### 推荐动作方式

1. 起点静止 2 $\sim$ 3 秒，让系统稳定
2. 缓慢直行
3. 缓慢转弯，不要急转
4. 尽量沿墙边走一圈
5. 回到起点形成回环

### 不推荐动作

- 猛加速
- 原地高速旋转
- 在狭窄区频繁来回抖动
- 地图还没稳定就保存

> [!CAUTION]
>
> Gmapping 特别依赖 “相邻两帧雷达之间变化不要太大”。机器人跑太快，本质上是在让 scan matching 做一道本不该它做的难题。

## 什么时候用 Gmapping，什么时候不用？

适合 Gmapping：

- 小型室内场地
- 2D 激光雷达
- 需要快速建图
- 算力一般

不太适合：

- 场景大且重复结构多
- 想要更稳的回环表现
- 要做长期运行

这时可考虑 Cartographer。

## 与 abot 比赛场景的关系

对教育机器人比赛来说，Gmapping 足够完成 “赛前快速扫一圈建图” 的任务。但想在比赛当天稳定复现，重点不在算法本身，而在：

1. 轮速计是否准确
2. 雷达数据是否干净
3. 机器人移动是否平稳
4. 参数是否偏安全

也就是说，**比赛里 Gmapping 更多是工程问题，而不是数学问题**。

---

## 建议的保守配置

如果你要为比赛重建地图，可以从下面这组值开始：

```xml
particles=20
minimumScore=50
delta=0.05
linearUpdate=0.05
angularUpdate=0.05
```

这组值比当前配置更稳，但地图会稍粗一些。对比赛导航通常是更好的取舍。
