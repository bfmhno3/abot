# 全局路径规划

> 从当前位置到目标点，先画一条“大致正确”的路

## 第一性原理

全局规划器不关心“下一秒电机怎么转”，它关心的是：

> “在整张地图上，从起点到终点，哪条路总体最合理？”

在 abot 中，全局规划器是：

```yaml
base_global_planner: global_planner/GlobalPlanner
```

它工作的基础是：

- 地图（`/map`）
- 当前定位（AMCL 提供的 `map → odom`）
- 目标点（例如 `/move_base_simple/goal`）

输出是一条全局路径：

- `/move_base/GlobalPlanner/plan`

这条路径通常是一条 “理想路线”，但不一定能直接执行——真正执行要靠 DWA 局部规划器。

## Dijkstra vs A*

当前配置：

```yaml
use_dijkstra: true
```

这意味着系统使用 **Dijkstra** 算法，而不是 A*。

### Dijkstra 的直觉

Dijkstra 会从起点向外 “灌水”，不断扩展最小代价区域，直到淹到目标点。

优点：

- 一定能找到最小总代价路径
- 比较稳

缺点：

- 对大地图来说比 A* 慢

### A* 的直觉

A* 会利用目标方向的启发信息，更有针对性地搜索。

优点：

- 更快

缺点：

- 对启发函数有依赖

对 abot 这种比赛小地图来说，Dijkstra 完全够用，而且更稳。

## 关键参数详解

### `use_quadratic = true`

表示在势场计算时使用二次近似，使路径更平滑一些。通常建议保持开启。

### `use_grid_path = false`

表示不是沿栅格边界 “折线式” 地走，而是使用梯度下降从势场中提取更自然的路径。

这会让全局路径看起来不那么 “像楼梯”。

### `allow_unknown = true`

表示允许规划穿过未知区域。

这在比赛中有两面性：

- **优点**：地图不完整时仍能规划
- **缺点**：机器人可能敢往没探明的地方走

如果你的地图建得比较完整，这个参数影响不大；如果地图缺块较多，它会直接决定机器人是否愿意 “穿未知区”。

### `default_tolerance = 0.5`

如果目标点恰好落在障碍物里或不可达，全局规划器会在目标附近 0.5m 半径内找一个最近可达点。

这很实用，因为比赛中手点目标或脚本给出的目标点不一定完美。

## 代价值参数

当前配置：

```yaml
lethal_cost: 253
neutral_cost: 50
cost_factor: 3.0
```

### `lethal_cost`

表示从 costmap 读取到多大代价值时，认为该栅格不可通过。

### `neutral_cost`

表示“普通可走区域”的基础代价。

### `cost_factor`

表示 costmap 中的代价对规划器有多大影响。

如果 `cost_factor` 调大，规划器会更害怕高代价区；调小，则更敢穿近障碍路径。

## 全局规划器在比赛中的真实作用

很多人会把导航问题都怪到 DWA 上，但其实全局规划先天也会决定路线性格：

- 全局路径太贴墙 → 局部规划器再好也容易危险
- 全局路径绕得离谱 → 局部规划只能跟着它绕
- 目标点本身不合理 → 全局规划无解

所以当你看到 “机器人为什么绕这么大一圈” 时，不要只看 DWA，也要看 GlobalPlanner 画出的全局路径。

---

## 什么时候要动全局规划参数？

通常全局规划参数不需要频繁改。优先改的是：

1. AMCL（先保证定位对）
2. costmap（先保证环境表达对）
3. DWA（再调局部行为）

只有当你发现：

- 路径整体形状就不对
- 经常规划不过去
- 地图明明通，规划器却绕很远

才应该回来检查 GlobalPlanner 参数。

---

## 推荐理解方式

把全局规划器看成“战略层”：

- 它决定大方向
- 不负责微操
- 不知道眼前的动态细节

局部规划器（DWA）则是“战术层”：

- 它负责把战略路线变成即时动作
- 它要处理近距离障碍和速度限制

这个分层理解非常关键，否则你会不断在错的地方调参数。
