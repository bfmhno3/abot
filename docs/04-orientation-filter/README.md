# 姿态滤波总览

> 从"陀螺仪会漂"到"稳定的朝向估计"

## 为什么需要姿态滤波？

IMU 的陀螺仪可以通过积分角速度得到航向角，但有一个致命问题：**漂移**。即使传感器静止，微小的零偏误差也会在积分过程中不断累积，几分钟后航向角就能偏几十度。

加速度计提供了一个**绝对参考**——重力方向永远向下。但它只提供倾斜信息（pitch/roll），无法确定航向（yaw）。而且加速度计受振动影响大，不能直接用来做姿态。

**姿态滤波的核心思想**：用陀螺仪提供高频、短时精确的角速度信息，用加速度计提供低频、长时稳定的重力方向参考，两者互补融合。

---

## 两种滤波算法

| 特性 | Mahony 互补滤波 | Madgwick 梯度下降 |
|------|-----------------|-------------------|
| 参数 | `twoKp` + `twoKi`（2 个） | `beta`（1 个） |
| 核心思想 | PI 控制器修正陀螺仪 | 梯度下降最小化重力误差 |
| 计算量 | 略低 | 略高 |
| 调参难度 | 中等（两个参数有交互） | 简单（单参数） |
| 积分项 | 有（消除稳态偏差） | 无 |
| 输出单位 | 角度（度） | 弧度 |

> [!TIP]
>
> 建议先尝试 Madgwick（单参数更易调），如果发现陀螺仪漂移严重（稳态偏差大），再尝试 Mahony（积分项可消除稳态偏差）。

## 输入/输出话题映射

```
输入（两种滤波器相同）：
  /imu/data_raw (sensor_msgs/Imu)

输出：
  /quaternion (geometry_msgs/QuaternionStamped)   — 姿态四元数
  /ypr (geometry_msgs/Vector3Stamped)             — 欧拉角（yaw/pitch/roll）
  TF: /odom → /imu                                — 坐标变换
```

> [!NOTE]
>
> 不要同时运行 Mahony 和 Madgwick 滤波器——它们发布相同的话题和 TF，会冲突。
