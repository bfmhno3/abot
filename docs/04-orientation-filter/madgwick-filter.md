# Madgwick 梯度下降滤波器

> 单参数，简单高效，入门首选

## 第一性原理：梯度下降优化

Madgwick 滤波器的核心思想可以类比为"下山"：

1. 定义一个**目标函数**：衡量当前姿态估计与加速度计测量之间的差异
2. 计算**梯度**：目标函数下降最快的方向
3. 沿梯度方向**走一步**：修正姿态估计
4. 步长由 **beta** 参数控制

这与机器学习中的梯度下降完全一样——只是优化的是姿态四元数，而不是神经网络权重。

---

## 与 Mahony 的对比

| 方面 | Mahony | Madgwick |
|------|--------|----------|
| 修正方式 | PI 控制器（比例+积分） | 梯度下降（只有"比例"） |
| 可调参数 | twoKp + twoKi | beta |
| 积分项 | 有（消除稳态零偏） | 无 |
| 计算复杂度 | 略低 | 略高（需要计算梯度） |
| 调参难度 | 中等 | 简单 |
| 稳态偏差消除 | 优秀（积分项） | 一般（无积分项） |

**选择建议**：

- 陀螺仪零偏大（静止时缓慢旋转）→ 选 Mahony（积分项消除零偏）
- 需要快速调参 → 选 Madgwick（单参数）
- 两者性能相近，在 abot 场景下差异不大

## 算法步骤详解

### 步骤 1：计算四元数导数（纯陀螺仪）

```cpp
// 从陀螺仪角速度计算四元数变化率
qDot1 = 0.5f * (-q1*gx - q2*gy - q3*gz);
qDot2 = 0.5f * ( q0*gx + q2*gz - q3*gy);
qDot3 = 0.5f * ( q0*gy - q1*gz + q3*gx);
qDot4 = 0.5f * ( q0*gz + q1*gy - q2*gx);
```

如果不做修正，仅用这个导数积分，就是纯陀螺仪积分——会漂移。

### 步骤 2：计算梯度（加速度计修正）

```cpp
// 目标函数的辅助变量（优化计算）
_2q0mx, _2q0my, _2q0mz, _2q1mx, ...  // 中间变量

// 梯度计算（6个方程，省略具体推导）
s0 = -_2q2 * (2.0f*q1q3 - _2q0q2 - ax) + ...
s1 =  _2q1 * (2.0f*q1q3 - _2q0q2 - ax) + ...
s2 = -4.0f*q2 * (1.0f - 2.0f*q1q1 - 2.0f*q2q2 - ay) + ...
s3 = -4.0f*q3 * (1.0f - 2.0f*q1q1 - 2.0f*q2q2 - az) + ...

// 归一化梯度
norm = sqrt(s0*s0 + s1*s1 + s2*s2 + s3*s3);
s0 /= norm; s1 /= norm; s2 /= norm; s3 /= norm;
```

梯度 s 指向 "**让姿态估计更接近加速度计测量**" 的方向。

### 步骤 3：应用修正

```cpp
// 从四元数导数中减去梯度方向
qDot1 -= beta * s0;
qDot2 -= beta * s1;
qDot3 -= beta * s2;
qDot4 -= beta * s3;
```

`beta` 控制修正的强度。

### 步骤 4：积分

```cpp
q0 += qDot1 * (1.0f / sampleFreq);
q1 += qDot2 * (1.0f / sampleFreq);
q2 += qDot3 * (1.0f / sampleFreq);
q3 += qDot4 * (1.0f / sampleFreq);

// 归一化
norm = sqrt(q0*q0 + q1*q1 + q2*q2 + q3*q3);
q0 /= norm; q1 /= norm; q2 /= norm; q3 /= norm;
```

## beta 参数详解

### 物理含义

`beta` 代表**陀螺仪测量误差的发散率**。它决定了滤波器对加速度计的信任程度：

- **beta 大**：更信任加速度计，快速修正但易受振动干扰
- **beta 小**：更信任陀螺仪，平滑但漂移慢修正

### 默认值：0.1088

这个值来自 Madgwick 的论文推导：

```
beta = sqrt(3/4) * gyro_noise_density  (rad/s)
```

对于典型的 MEMS 陀螺仪，计算结果约为 0.04 $\sim$ 0.15。0.1088 是一个中间值。

### 调参指南

| 场景 | 建议 beta 值 | 理由 |
|------|-------------|------|
| 机器人静止/缓慢移动 | 0.1~0.2 | 可以多信任加速度计 |
| 快速运动/振动大 | 0.02~0.05 | 减少加速度计噪声影响 |
| 陀螺仪质量好 | 0.01~0.05 | 可以多信任陀螺仪 |
| 陀螺仪漂移严重 | 0.1~0.3 | 需要加速度计快速修正 |

### 调参步骤

1. 从 `beta=0.1` 开始
2. 静止放置机器人，观察 `/ypr` 输出是否稳定
3. 如果震荡 → 减小 `beta`
4. 如果缓慢漂移 → 增大 `beta`
5. 快速旋转后静止，观察恢复时间
6. 通过 `rqt_reconfigure` 实时调整

## 输出单位

> [!WARNING]
>
> Madgwick 滤波器的 `/ypr` 话题输出**弧度**，而 Mahony 输出**角度（度）**。如果在代码中混用，必须注意单位转换。

```cpp
// Madgwick 输出弧度
yaw = atan2(2*q1*q2 - 2*q0*q3, 2*q0*q0 + 2*q1*q1 - 1);
// 不乘 57.29578

// Mahony 输出角度
yaw = atan2(...) * 57.29578;  // 乘以 180/π
```

如果需要统一，可以修改源码或在下游节点中转换。

## 动态参数配置中的错误描述

```python
# 源码：src/imu_filter/cfg/MyStuff2.cfg
gen.add("beta", double, 0, "2* accelerometer proportional gain", 0.1088, 0, 0.5)
```

> [!NOTE]
>
> 描述文字 `"2* accelerometer proportional gain"` 是错误的。`beta` 是 Madgwick 滤波器的增益参数，代表陀螺仪误差发散率，与 "2* 加速度计比例增益" 无关。这是从 Mahony 的参数描述复制过来的遗留错误，不影响功能。
